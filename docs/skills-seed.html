Local Skills for Claude Code and Codex

让 AI Agent 先理解项目规则，再开始改代码。

Skills Seed 从已有代码、Git 历史和检查命中里提取团队真实实践，生成本地可加载的 Skills，让 Agent 按你的项目边界、业务入口和测试习惯工作。

两步开始 查看命令参考
skills-seed 
# 在已有 Git 项目根目录执行
cd your-project
skills-seed init
skills-seed sync

# Codex 默认入口
.agents/skills/skills-seed-skills/SKILL.md

两个命令跑完整个流程

init 和 sync 都有交互式界面。日常使用者不需要记住底层学习、生成或整理命令。

1

初始化

运行 skills-seed init，选择单项目或 workspace、Agent、语言和执行计划。

2

同步

运行 skills-seed sync，学习当前代码，有变化时自动刷新 Skills。

3

使用

打开 Claude Code 或 Codex，Agent 会按生成的 SKILL.md 读取项目上下文。

选择适合你的使用方式

保持首页足够简单，同时覆盖单项目、workspace 和自动化接入三类主路径。

使用方式 适合场景 入口
交互式单项目 第一次给已有项目生成 Skills。 skills-seed init → skills-seed sync
Workspace 根目录下有多个独立 Git 子项目。 在 init 中选择 workspace，再执行 sync
CI / 脚本 团队模板或自动化环境中不适合交互。 --no-interactive、--resume

沉淀真实项目经验

Skills Seed 面向已有项目，不要求你先写一份完整规范。它从仓库事实中提取规则，再允许团队补充长期约束。

项目规则学习

从当前代码和 Git 历史中提取 patterns、业务方法、工具方法、测试策略和项目画像。

Agent 可加载产物

生成 SKILL.md 和 references，让 Claude Code / Codex 按任务读取最小必要上下文。

Workspace 路由

根仓负责路由和跨项目关系，子项目独立学习，避免上下文互相污染。

变更检查

用已学习规则检查当前改动，记录 pattern hits，让真正有用的规则更容易进入最终 Skills。

人工规则补充

通过 patterns 和 workflow 补充团队长期规则和常用任务流程。

本地优先

配置、数据库、运行日志和生成产物默认保存在当前仓库内，不维护远端知识库。

生成后 Agent 会知道什么

它不是普通项目简介，而是围绕当前仓库组织的工作上下文。

目录与模块边界

哪些目录负责什么能力，改动时应该先看哪里。

业务与工具入口

团队长期使用的业务方法、工具方法、错误处理和测试习惯。

Workspace 路由

根仓中哪个子项目处理哪类需求，跨项目改动按什么顺序看。

不是又一份手写说明

手写项目说明很容易过期。Skills Seed 的重点是从已有仓库事实中持续提取和刷新 Agent 上下文。

增量更新

项目变化后运行 sync，学习结果随当前仓库更新。

保留结构

入口文档保持简洁，详细画像、业务入口和模式证据进入 references。

反馈驱动

check 和 review hits 会记录规则命中，帮助判断哪些规则真正有用。

主命令索引

README 只列常用主命令；子命令和参数由命令参考维护，避免用户在首页被细节淹没。

命令 用途
skills-seed init交互式初始化当前项目或 workspace。
skills-seed sync交互式学习当前代码并刷新 Skills。
skills-seed workspace管理 workspace 子项目。
skills-seed patterns查看、补充、修订或整理模式规则。
skills-seed workflow添加或更新团队常用任务流程。
skills-seed check用已学习规则检查当前改动。
skills-seed hook安装、卸载或手动运行 Git hook。
skills-seed profile查看或刷新项目画像。
skills-seed preview预览同步时可能纳入的文件范围。
skills-seed log查看学习和生成变更记录。

生成的 Skills 结构

SKILL.md 是 Agent 入口，references 保存更完整的项目画像、规范、业务入口和模式细节。

这种结构让 Agent 先读取入口，再按任务深入相关参考文件，而不是把所有规则一次性塞进上下文。

.agents/skills/skills-seed-skills/ ├── SKILL.md ├── agents/ │ └── openai.yaml └── references/ ├── project-overview.md ├── project-spec.md ├── business-methods.md ├── modules.md └── patterns/ ├── business.md ├── error.md └── testing.md

Workspace 友好

一个根目录下有多个独立 Git 子项目时，选择 workspace 模式。根仓负责编排、路由和跨项目关系，子项目独立保存自己的学习结果。

workspace 
cd your-workspace
skills-seed init
skills-seed sync

# 新增子项目后使用 workspace 主命令管理
skills-seed workspace --help
