Local skills for Claude Code and Codex

让 AI Agent 先理解项目规则，再开始改代码。

Skills Seed 从已有代码、Git 历史和检查命中里提取团队真实实践，生成本地可加载的 Skills，让 Agent 按你的项目边界、业务入口和测试习惯工作。

两步开始 查看命令参考
skills-seed
# 在已有 Git 项目根目录执行 cd your-project skills-seed init skills-seed sync
SKILL.md Agent 入口
project-overview.md project-spec.md business-methods.md patterns/*.md

Workflow

两个命令跑完整个流程

init 和 sync 都会根据当前仓库状态动态切换：首次运行进入配置流程，再次运行会提供查看、重置、续跑或重新同步。

01

初始化

首次运行选择单项目或 workspace、Agent、语言和执行计划；再次运行可查看当前配置或重新初始化。

02

同步

首次同步建立第一版上下文；已有产物后执行增量学习；存在未完成状态时可续跑或重来。

03

使用

打开 Claude Code 或 Codex，Agent 会按生成的入口文件读取最小必要项目上下文。

What it captures

沉淀真实项目经验

它面向已有项目，不要求你先手写完整规范。仓库事实、团队补充和检查命中会一起形成可持续刷新的 Agent 上下文。

项目规则学习

从当前代码和 Git 历史中提取 patterns、业务方法、工具方法、测试策略和项目画像。

Agent 可加载产物

生成 SKILL.md 和 references，让 Claude Code / Codex 按任务读取最小必要上下文。

Workspace 路由

根仓负责路由和跨项目关系，子项目独立学习，避免上下文互相污染。

变更检查

用已学习规则检查当前改动，记录 pattern hits，让真正有用的规则更容易进入最终 Skills。

Generated output

生成的 Skills 结构

SKILL.md 是 Agent 入口，references 保存更完整的项目画像、规范、业务入口和模式细节。Agent 先读入口，再按任务深入相关参考文件。

CLI

主命令索引

首页保留常用入口；完整子命令和参数由命令参考维护。

skills-seed init交互式初始化当前项目或 workspace。
skills-seed sync交互式学习当前代码并刷新 Skills。
skills-seed workspace管理 workspace 子项目。
skills-seed patterns查看、补充、修订或整理模式规则。
skills-seed workflow添加或更新团队常用任务流程。
skills-seed check用已学习规则检查当前改动。

Workspace

多项目仓库也能保持上下文边界

一个根目录下有多个独立 Git 子项目时，选择 workspace 模式。根仓负责编排、路由和跨项目关系，子项目独立保存自己的学习结果。

cd your-workspace
skills-seed init
skills-seed sync

# workspace
skills-seed workspace --help
